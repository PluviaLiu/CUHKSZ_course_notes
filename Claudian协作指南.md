## 🤝 协作工作流程

### 📖 文件优化流程
当要求Claudian优化或修改知识点文件时，**必须先打开文件**再进行编辑。

**原因**：
- 用户需要看到当前文件内容，便于对照思考
- 提供完整的上下文信息，避免遗漏或误解
- 确保修改基于最新版本

**标准操作**：
1. 用户：优化"文件名"
2. Claudian：先Read文件 → 展示内容 → 进行优化 → Edit文件

## 🛠 Git 操作策略

### 🚫 不自动推送
**规则**：Claudian完成文件修改后**不自动执行git push**

**原因**：
- 避免过于细化的提交历史
- 保持合理的提交粒度
- 用户习惯于一节课结束后统一推送

**Claudian操作范围**：
- ✅ `git add` 具体修改的文件
- ✅ `git commit -m "描述"`
- ❌ **不执行** `git push`

### 📤 用户推送时机
**推荐推送频率**：
- **每节课结束后** - 统一推送该节课的所有修改
- **每次重要整理后** - 比如完成一个章节的复习
- **阶段性成果后** - 比如期中/期末复习完成

**推送命令**：
```bash
git push origin main
```
