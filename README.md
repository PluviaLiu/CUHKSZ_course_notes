## 🧠 笔记方法：原子化 + 双链

我的日常整理遵循“**原子化笔记**”思路：

1. **解题 / 阅读**：从题目或讲义出发。
2. **拆解**：遇到关键知识点时，用 `[[知识点名称]]` 建立双链。
3. **沉淀**：跳转到该独立条目，记录定义、定理、证明、例题与常见坑。
4. **回归**：回到原题/原笔记继续推进。

这样能让知识点跨课程复用，并依赖反向链接/图谱快速复习。

## 🛠 Obsidian 配置（可选）

阅读笔记不强依赖这些配置；如果你也用 Obsidian，下面设置能获得类似VScode的UI界面，与完善的AI协助体验。

### 1) PDF 相关
- **Text Extractor**：提取 PDF 文本（便于检索/AI 工具读取）  
  用法：打开 PDF → 右上角菜单 → 执行提取。

### 2) 版本控制
- **Terminal**（插件）：在 Obsidian 内执行 Git 命令
  - `git add .`
  - `git commit -m "..." `
  - `git push`

### 3) 外观设置
- 主题：**Blue Topaz**
- 字体：**KXGW WenKai**
- **Stille**：调暗除光标所在段落外的内容，辅助专注

### 4) AI 辅助
- 插件：**Copilot**
- 模型：**Gemini**
- 理由：对 LaTeX 公式复制/渲染更友好，并支持图片提问（适合题目截图询问）。
- 用法：可在 Copilot Commands 里预设 Prompt；输入框 `/` 或选中文本右键调用。

#### Claude/Claudian 支持
- **Claudian**：将Claude code内接到Obsidian内
  - **适用场景**:
    - 批量化处理：“帮我给这一系列标题重命名为Def/Thm/Lemma等开头”
    - 双链处理：“给我扫描这个知识库下的知识点，在知识点的地方建立双链

### 5) 任务管理
- 插件：**Tasks**
- 我通常在文档开头记录任务与使用日期。之后，建立统一的任务看板，就可以纵览需要修改的文件和待整理的知识点。
